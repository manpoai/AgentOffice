# Contributing

Thanks for your interest in AgentOffice.

## Before opening a PR

Please first check:
- existing issues
- current docs
- whether the change fits the current release direction

## What contributions are welcome

- bug fixes
- documentation improvements
- install / self-hosting fixes
- agent integration improvements
- editor reliability fixes

## Development note

AgentOffice currently contains multiple subprojects:
- `shell/`
- `gateway/`
- `mcp-server/`
- `adapters/`

Please keep changes scoped and explain which part you touched.

## Pull requests

A good PR should include:
- what changed
- why it changed
- how to test it
- whether it affects release/install behavior
