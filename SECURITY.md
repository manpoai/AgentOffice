# Security Policy

If you believe you found a security issue in aose, please do not open a public issue with exploit details.

## Please include

- affected component
- reproduction steps
- impact
- whether the issue can expose data, tokens, or unauthorized access

## Current guidance

Until a dedicated security channel is finalized, report security issues privately to the maintainers through GitHub private coordination where possible.

Public issues are fine for general hardening discussions, but not for active exploit details.
